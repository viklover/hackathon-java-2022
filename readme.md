Задание на хакатон
==================
В результате работы системы ИИ по распознаванию документов генерируется отчет-реестр распознанных данных.<br>
Реестр представляет собой CSV-файл - структурированный текстовый файл (кодировка Windows 1251), разделитель запятая.
В каждой строке реестра перечисляется список распознанных значений полей соответствующего документа.<br>
Существует несколько типов документов, соответственно структура реестра распознанных значений будет различаться.
В реестре также содержатся имена файлов-оригиналов (pdf) – сканов документов, по которым проведено распознание данных.

Задача
------
Разработать решение, позволяющее оператору<br>
!! Пункты выделенные *курсивом* - дополнительные задания

1. Загрузить полученный файл-реестр в базу данных
  1. Дубликаты существующих записей игнорируются
  1. *Выбор: игнорировать или перезаписать дубликаты существующих записей*
2. В зависимости от выбранного типа документа получить соответствующий список (реестр) распознанных данных в виде таблицы 
  1. Для каждой строчки таблицы реестра иметь возможность:
  1. Пометить строку как __проверенную__
  1. Выбрать любой элемент строки и внести изменения значения. При этом:
     - Элемент и строка помечаются как __измененные__
     - Доступен просмотр оригинального (распознанного) значения
Данную операцию можно проводить несколько раз, при этом сохраняем только оригинальное распознанное значение и последний вариант исправления
  3. Посмотреть оригинал скана документа, соответствующего данной строке реестра
  3. Снять пометку __проверено__ – при этом информация о редактированных элементах строки сохраняется
4. При повторном подключении к странице / обновлении страницы вся информация должна сохраняться 
5. *Просмотреть статистику по полям/типам документов в разрезе частоты ошибок распознавания*
6. *Предусмотреть выгрузку строк реестра, помеченных как Проверено в виде csv-файла (кодировка Windows-1251)*
  1. *Для элементов с измененными значениями – выгружать только последний вариант исправлений*
  2. *Выбор: все проверенные строки или только проверенные и с изменениями* 

Функциональные требования:
--------------------------
1. Предусмотреть большое (1000+) количество документов
2. *Предусмотреть возможность одновременной работы нескольких операторов (блокировка одновременного редактирования)*

Технические требования 
----------------------
1.Архитектура решения: Front-END + Back-END + DataBase
  1. Front-END: HTML/CSS/JS. Любые открытые фреймворки
  2. Back-END: Java/Node.js или любая другая платформа. Без ORM, только ванильный SQL
  3. База данных: MySQL
  4. Рекомендованный стандарт обмена данными клиент-сервер __JSON__

Оформление решения
------------------
1. Страница 
2. Документация 
  1. Описание структуры базы данных:
     - Наименование таблицы, перечень полей, предназначение
     - Связи между таблицами 
  2. Описание Back-END решения: 
     - Выбранная платформа
     - Описание структуры классов и их назначение 
     - Описание API – формата запроса от клиента и формата ответа от сервера
  3. Описание Front-END решения:
     - Описание выбранных фреймворков
     - Инструкция пользователя
3. Для базы данных создать скрипт: 
   - Создания пустых таблиц и связей между ними
